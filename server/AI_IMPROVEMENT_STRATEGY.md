```

```

### 1. Redis 캐싱 레이어 ⭐⭐⭐

#### 현재 문제

```
DB 조회: 0.5초
→ 사용자 경험 저하
→ DB 부하
```

#### 개선 방안

```typescript
// 3단계 캐시 전략

1. Redis 확인 (0.01초)
   - 캐시 히트: 즉시 반환
   - 캐시 미스: 다음 단계

2. DB 조회 (0.5초)
   - DB에 있으면: Redis에 캐시 + 반환
   - 없으면: 다음 단계

3. AI 생성 (3초)
   - 새로 생성
   - DB 저장
   - Redis 캐시

// 효과:
// - 95% 캐시 적중 시: 평균 0.05초
// - 기존 0.5초 대비 10배 향상
```

#### 구현 완료

```
✅ CacheModule (캐시 모듈)
✅ CacheService (메모리 캐시 → Railway Redis 준비됨)
✅ PredictionCacheService (3단계 캐시 로직)
```

```typescript
// 전략적 모델 선택
```

```typescript
// AIBatchService에 적용

async batchPredictTodayRaces() {
  for (const race of races) {
    // 최적 모델 선택
    const model = await this.costOptimizer.selectOptimalModel({
      raceId: race.id,
      raceGrade: race.rcGrade,
      racePrize: race.rcPrize,
    });

    // 선택된 모델로 예측
    const prediction = await this.predictionsService.generatePrediction({
      raceId: race.id,
      llmProvider: model.includes('gpt') ? 'openai' : 'anthropic',
      modelVersion: model,
    });
  }
}
```
