# 📊 데이터 수집 가이드

## 🎯 목적

Golden Race AI 모델 학습을 위해 한국마사회(KRA) API로부터 과거 경마 데이터를 수집합니다.

## 🚀 빠른 시작

### 1단계: 환경 준비

```bash
cd server

# MySQL 시작
docker-compose up -d mysql

# 서버 의존성 설치 (아직 안했다면)
npm install
```

### 2단계: 현재 데이터 확인

```bash
npm run init:data:check
```

**출력 예시:**

```
📊 데이터베이스 커버리지 확인

🏁 경주 데이터: 0건
📊 경주 결과: 0건
📋 경주 계획: 0건

📅 데이터 기간: (데이터 없음)
```

### 3단계: 초기 데이터 수집

```bash
# 권장: 최근 30일로 시작
npm run init:data:30days
```

## 📋 수집 전략

### AI 모델 학습용 (권장)

#### 최소 데이터셋

```bash
# 1년치 데이터 (~18,000 경주)
npm run init:data:year
```

#### 권장 데이터셋

```bash
# 2024년 전체
npm run init:data -- --year=2024

# 2023년 전체
npm run init:data -- --year=2023

# 2022년 전체
npm run init:data -- --year=2022
```

#### 대규모 데이터셋 (최고 성능)

```bash
# 3년치 데이터 (~54,000 경주)
npm run init:data -- --start=20220101 --end=20241231
```

### 테스트 및 개발용

#### 샘플 데이터

```bash
# 최근 7일
npm run init:data -- --days=7
```

#### 특정 경주장 집중 수집

```bash
# 서울 경주장 집중 분석용 (별도 필터링 필요)
npm run init:data -- --start=20240101 --end=20240131
```

## ⏱️ 예상 소요 시간

### API 호출 제한 고려

- **KRA API 제한**: 분당 100건, 일일 10,000건
- **수집 딜레이**: 요청 간 1초
- **하루 데이터**: 약 50-70초 (경주 수에 따라 다름)

### 기간별 예상 시간

| 기간 | 일수  | 예상 경주 | 소요 시간 |
| ---- | ----- | --------- | --------- |
| 7일  | 7     | ~350      | ~7분      |
| 30일 | 30    | ~1,500    | ~30분     |
| 90일 | 90    | ~4,500    | ~1.5시간  |
| 1년  | 365   | ~18,000   | ~6시간    |
| 3년  | 1,095 | ~54,000   | ~18시간   |

**💡 팁**: 대량 수집은 밤에 실행하는 것을 권장합니다!

## 🗄️ 데이터 구조

### 수집되는 테이블

```sql
-- 경주 결과
results: 경주 완료 후 순위, 기록 등

-- 경주 계획
race_plans: 예정된 경주 정보

-- 경주 기본 정보
races: 경주 메타데이터

-- 배당율
dividend_rates: 확정 배당률

-- 출전표 (통계만)
entry_details: 참가 마필 상세 정보
```

### 데이터 품질 체크

```bash
# 수집 후 검증
npm run init:data:check
```

**확인 항목:**

- ✅ 데이터 개수
- ✅ 날짜 범위
- ✅ 중복 없음
- ✅ 완전성

## 🔄 증분 업데이트

### 일일 자동 업데이트

배치 스케줄러가 자동으로 실행됩니다:

- **오전 6시**: 전날 경주 결과
- **오전 7시**: 오늘 경주 계획
- **오후 2시**: 확정 배당율

### 수동 업데이트

```bash
# 누락된 데이터 보완
npm run init:data -- --start=20240515 --end=20240520
```

## 📊 AI 학습용 데이터 준비

### 데이터 분할 전략

#### 학습 데이터 (70%)

```
2022-01-01 ~ 2023-12-31 (2년)
약 36,000 경주
```

#### 검증 데이터 (15%)

```
2024-01-01 ~ 2024-06-30 (6개월)
약 9,000 경주
```

#### 테스트 데이터 (15%)

```
2024-07-01 ~ 2024-12-31 (6개월)
약 9,000 경주
```

### 수집 순서

```bash
# 1. 학습 데이터
npm run init:data -- --start=20220101 --end=20231231

# 2. 검증 데이터
npm run init:data -- --start=20240101 --end=20240630

# 3. 테스트 데이터
npm run init:data -- --start=20240701 --end=20241231

# 4. 최종 확인
npm run init:data:check
```

## 🐛 디버깅

### 상세 로그 보기

NestJS 로거가 자동으로 상세 로그를 출력합니다.

### 특정 날짜만 테스트

```bash
# 하루치만 수집 (테스트용)
npm run init:data -- --start=20240901 --end=20240901
```

### 에러 발생 시

- 에러가 발생한 날짜는 기록되고 계속 진행됩니다
- 완료 후 요약에서 실패한 날짜 확인 가능
- 실패한 날짜만 다시 수집 가능
