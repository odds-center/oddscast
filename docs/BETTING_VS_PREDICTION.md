# 베팅 기록 vs AI 예측 시스템

## 🎯 핵심 개념

Golden Race는 **두 가지 시스템**을 제공합니다:

### 1. 베팅 기록 관리 📝

**목적**: 사용자가 실제 합법적 경마 사이트에서 구매한 마권을 기록하고 추적

#### 특징

- ✅ **기록만 제공**: 앱 내에서 실제 베팅 불가
- ✅ **외부 베팅 추적**: 한국마사회 등에서 구매한 마권 정보 입력
- ✅ **자동 결과 확인**: 경주 종료 시 자동으로 승패 판정
- ✅ **통계 제공**: 개인 베팅 히스토리 및 수익률 분석
- ❌ **앱 내 결제 없음**: 포인트나 현금으로 베팅 불가

#### 사용 흐름

```
1. 사용자가 한국마사회 사이트에서 마권 구매
   ↓
2. Golden Race 앱에 베팅 정보 입력
   - 경주: 서울 1R
   - 마권 종류: 단승식
   - 선택 말: 3번 천둥번개
   - 구매 금액: 5,000원 (기록용)
   ↓
3. 경주 종료 후 자동 결과 확인
   ↓
4. 승패 여부 및 배당금 표시
   ↓
5. 통계에 반영 (승률, ROI 등)
```

### 2. AI 예측 게임 🤖

**목적**: 포인트를 사용하여 AI와 예측 경쟁을 하는 게임

#### 특징

- ✅ **게임 내 포인트**: 가상 화폐로 예측 참여
- ✅ **AI 비교**: AI 예측과 사용자 예측 비교
- ✅ **랭킹 시스템**: 정확도 기반 순위
- ✅ **포인트 획득**: 정확한 예측 시 포인트 획득
- ❌ **현금 전환 불가**: 포인트는 게임 내에서만 유효

#### 사용 흐름

```
1. 경주 선택
   ↓
2. AI 예측 확인 (선택사항, 포인트 사용)
   ↓
3. 사용자 예측 입력 (포인트 사용)
   - 1위: 3번
   - 2위: 5번
   - 3위: 1번
   ↓
4. 경주 종료 후 결과 확인
   ↓
5. AI vs 사용자 정확도 비교
   ↓
6. 정확도에 따라 포인트 획득
   ↓
7. 랭킹 업데이트
```

## 🗂️ 데이터베이스 구조

### Bets 테이블 (베팅 기록)

```sql
CREATE TABLE bets (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36),
    race_id VARCHAR(50),
    bet_type ENUM(...),
    bet_amount DECIMAL(15,2),        -- 실제 구매 금액 (기록용)
    actual_win DECIMAL(15,2),        -- 실제 당첨금 (기록용)
    bet_source VARCHAR(20),          -- 'KRA', 'EXTERNAL' (구매처)
    is_registered BOOLEAN,           -- 외부 구매 기록 여부
    ...
);
```

### Predictions 테이블 (AI 예측 게임) - 신규 생성 필요

```sql
CREATE TABLE predictions (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36),
    race_id VARCHAR(50),
    predicted_winner VARCHAR(20),    -- 예측한 1위
    predicted_top3 JSON,             -- 예측한 상위 3위
    points_used INT,                 -- 사용한 포인트
    points_earned INT,               -- 획득한 포인트
    accuracy_score DECIMAL(5,2),     -- 정확도 점수
    ai_prediction JSON,              -- AI 예측 (비교용)
    prediction_time DATETIME,
    result_time DATETIME,
    status ENUM('PENDING', 'COMPLETED'),
    ...
);
```

## 🔄 수정 방향

### 즉시 수정 필요

1. **용어 정리**

   - "베팅" → 외부 마권 **기록** (명확히 구분)
   - "예측" → 게임 내 AI **예측 게임**

2. **UI 분리**

   - 베팅 기록 탭: 외부 구매 마권 관리
   - 예측 게임 탭: AI 예측 경쟁

3. **Predictions 시스템 구축**
   - 새로운 엔티티/서비스/컨트롤러
   - AI 예측 API 통합
   - 포인트 기반 게임 로직

### 법적 안전장치

- 베팅 기록 화면에 명시: "외부에서 구매한 마권을 기록하는 기능입니다"
- 예측 게임에 명시: "게임 내 포인트를 사용하는 예측 게임입니다"
- 앱 내에서는 절대 실제 마권 구매 불가
