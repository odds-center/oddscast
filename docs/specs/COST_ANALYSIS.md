# 💰 Gemini API 비용 분석 & 서버 캐싱 전략

> 이 문서는 Gemini API 비용과 서버 캐싱을 통한 비용 절감 전략을 정리합니다.

---

## 핵심 결론

- **초기 런칭:** 0원 (무료) 운영 가능
- **사용자 증가 후:** 월 $5 미만 (커피 한 잔 값)
- **핵심 원리:** 서버가 미리 분석(Caching) → 사용자는 DB 읽기만

---

## 1. Gemini API 요금 정책 (2025년 기준)

**Gemini 1.5 Flash** (가성비 모델, 속도 빠름) 사용 권장.

### Free Tier (무료 구간)

| 항목      | 제한                |
| --------- | ------------------- |
| 분당 요청 | 15회 (15 RPM)       |
| 하루 요청 | 1,500회 (1,500 RPD) |
| 비용      | **$0**              |

### Pay-as-you-go (유료 구간)

| 항목            | 비용                         |
| --------------- | ---------------------------- |
| 입력 (Prompt)   | 100만 토큰당 $0.075 (~100원) |
| 출력 (Response) | 100만 토큰당 $0.30 (~400원)  |

---

## 2. 비용 시뮬레이션

### 시뮬레이션 조건

- **경기 수:** 한국 경마 (금/토/일) 주 50경기 × 4주 = **월 200경기**
- **분석 빈도:** 경기당 3회 (출전표 확정, 배당 변경, 마감 직전) = **월 600회**
- **토큰 사용량 (1회):**
  - 입력: ~1,500 토큰
  - 출력: ~500 토큰
  - 1회 비용: ~$0.0003 (0.4원)

### 시나리오 A: 무료 티어 (추천 ✅)

- 하루 요청: ~60회 (주말 기준)
- Google 무료 제공: 하루 1,500회
- **결과: 100% 무료 운영**
- 사용자 1만 명이 와도, 서버는 DB 저장된 분석글만 제공 → API 호출 불변

### 시나리오 B: 유료 구간 (채팅 기능 추가 시)

- 월 10,000건 질문 발생 가정
- 입력 토큰: 1,500만 / 출력 토큰: 500만
- **총합: 월 $2.62 (~3,500원)**

## 3. 비용 절감 아키텍처 핵심 전략

> 이 원칙을 지키면 **사용자 100만 명이 되어도 API 비용은 동일**합니다.

### 1) Server-Side Caching (서버 캐싱) ⭐ 핵심

```
❌ 사용자 요청 → 매번 Gemini 호출 (비용 폭발)
✅ 스케줄러(cron) → Gemini 호출 → DB 저장 → 사용자는 DB만 읽음 (비용 고정)
```

- 사용자가 앱을 켤 때 Gemini를 부르지 않음
- 서버가 스케줄러로 Gemini에게 물어보고, `Analysis` 테이블에 저장
- 사용자는 DB에 저장된 텍스트만 조회 (Read DB Cost만 발생)

### 2) 모델 선택

| 모델                 | 용도                               | 비용         |
| -------------------- | ---------------------------------- | ------------ |
| **Gemini 1.5 Flash** | 승부 예측 분석 (추천)              | 저렴         |
| Gemini 1.5 Pro       | 프리미엄 분석 리포트 (유료 사용자) | 10~20배 비쌈 |

### 3) 프롬프트 최적화

- Python 계산 결과값(숫자)만 JSON으로 전달
- 불필요한 서술형 텍스트 제거 → 입력 토큰 비용 절감
