## 1. WebApp (유저 사이트) 부족한 면

### 1.1 데이터·로딩·에러

| 항목 | 상태 | 비고 |
|------|------|------|
| 일부 목록/상세 에러 시 "다시 시도" 누락 | 적용됨 | 주요 페이지 DataFetchState·에러 시 다시 시도 적용 완료 |
| 빈 상태(Empty) 문구 통일 | 적용됨 | DataFetchState·emptyTitle "~가/이 없습니다" 패턴으로 페이지별 적용 |
| 대용량 페이지 데이터 경고 | 완화 적용 | 홈 prefetch limit 축소(results 60→30, races 50/150→30/40), 매트릭스 중복 prefetch 제거 |

### 1.2 UX·콘텐츠

| 항목 | 상태 | 비고 |
|------|------|------|
| 일일 로그인 보너스 | 적용됨 | 로그인 시 1일 1회 포인트 지급, 로그인 페이지에서 안내 메시지 |
| 연속 로그인 보상 (7일 연속 등) | 적용됨 | 7일 연속 시 RACE 티켓 1장, 프로필에 연속 일수 표시 |
| AI 신뢰도 표시 | 적용됨 | 경주 상세 예측에 신뢰도(높음/보통/낮음) + winProb % 표시 |
| 푸시 타이밍 개선 | 적용됨 | 첫 경주 30분 전 알림 (서버 크론, raceEnabled 대상) |
| 이미지 최적화 | 선택 | 경주/말 사진 추가 시 next/image + alt (FEATURE_ROADMAP §5.4) |

### 1.3 모바일·접근성

| 항목 | 상태 | 비고 |
|------|------|------|
| **모바일 첫 화면** | 적용됨 | 로그인 시 홈(/) 유지, 비로그인 시 로그인 페이지(/)가 첫 화면. WebApp _app에서 네이티브일 때만 리다이렉트 |
| Push Deep Link (모바일) | 적용됨 | 알림 탭 시 deepLink로 WebView 초기 URL 설정 (index + webview 연동) |
| 터치 영역·버튼 크기 | 적용됨 | 44px 최소(버튼·FilterChips·TabBar·BackLink), touch-manipulation 적용 |
| 고대비/글자 크기 기본값 | 적용됨 | 설정에서 토글 가능 |
| **AppBar 스크롤 hide/show** | 적용됨 | 스크롤 다운 시 하단 AppBar slideDown, 스크롤 업 시 slideUp. px padding 추가. 투명도 제거(solid white). |
| **모바일 버튼 레이아웃** | 적용됨 | 홈/구독/매트릭스 등 전 영역 flex-col 스택 레이아웃. space-between 제거. w-full 버튼. |
| **출전마 번호(chulNo) 표시** | 적용됨 | HorseEntryTable·ResultCard·BetTypePredictionsSection·HorseScoresBarChart·PredictionMatrixTable·HorsePickPanel 모두 circle badge 표시 |
| **개별/종합 예측권 분리 표기** | 적용됨 | 프로필 4컬럼(개별권·종합권·포인트·구독), 경주 상세에서 개별권만 표시 |

### 1.4 페이지별 보강 (우선 점검 권장)

| 페이지 | 부족할 수 있는 부분 |
|--------|----------------------|
| 홈 | 오늘 경기 종료 문구 통일(TODAY_ALL_ENDED_MESSAGE), 섹션별 에러 시 refetch 적용됨 |
| 경주 목록/상세 | 필터·URL 동기화 적용됨, 결과/에러 문구 정리 적용됨 |
| 결과 목록 | 날짜·경마장 필터·URL 동기화 적용됨, 빈/에러 문구 적용됨 |
| 종합 예측(매트릭스) | 날짜 변경 시 로딩·에러·첫 방문 안내 적용됨 |
| 마이페이지(구독·포인트·티켓) | 결제 성공/실패 안내·만료 티켓 안내 적용됨 |
| 프로필·설정 | 알림 설정 저장 시 "저장되었습니다" 인라인 피드백 적용됨, 회원탈퇴 플로우 |
| 말/기수/조교사 프로필 | DataFetchState 로딩·에러·빈 상태 적용됨 |

## 2. Admin (관리자) 부족한 면

### 2.1 공통

| 항목 | 상태 | 비고 |
|------|------|------|
| 로딩/에러/빈 상태 통일 | 적용됨 | 경주 목록·경기 결과 페이지 에러 시 메시지 + 다시 시도 버튼 추가 |
| 날짜 필터 KST | 적용됨 | 경주 목록·경기 결과·KRA 동기화 날짜 기본값 KST 오늘 적용 |
| 페이지별 권한/가드 | 적용됨 | Layout에서 useRequireAuth 훅으로 토큰 검사, 미로그인 시 /login 리다이렉트 (401 인터셉터와 이중 적용) |

### 2.2 경주(KRA)·동기화

| 항목 | 상태 | 비고 |
|------|------|------|
| 경주 목록 날짜 기본값 | 수정됨 | 기본값을 빈 문자열(전체 경주 표시)로 변경. 오늘 날짜 기본값은 경주 없는 평일에 0건 표시 문제 유발. 필터 초기화 버튼 양쪽 뷰에서 표시. |
| 동기화 진행률·로그 | 적용됨 | SyncProgressBar, 토스트 메시지. syncAnalysisData 세부 진행률 추가. |
| 연도별 전체 적재 | 적용됨 | POST /api/admin/kra/sync/year-stream — 경주계획+과거 출전표·결과·배당률·상세정보 SSE 스트리밍 |
| NULL 덮어쓰기 방지 | 적용됨 | 6곳 upsert/update에서 conditional spread 적용. 배당률 실패 시 warnings 배열로 전달 |
| 결과 동기화 후 상태값 | 적용됨 | COMPLETED는 실제 결과/착순 반영 후만 (서버 수정 반영) |
| 배치 스케줄 요약 화면 | 적용됨 | Admin KRA 페이지에 배치 스케줄(예정/완료/실패) 테이블·집계 표시, 15초마다 갱신 |

### 2.3 사용자·구독·통계

| 항목 | 상태 | 비고 |
|------|------|------|
| 사용자 목록 검색·필터 | 적용됨 | 이메일·사용자명 검색 + 에러 시 다시 시도 |
| 구독/매출 대시보드 | 적용됨 | revenue, statistics 등 |
| 예측 정확도·티켓 사용량 | 적용됨 | 예측 목록 에러 재시도, 경주별 적중률 링크(경주 상세), 배치 ON/OFF·오늘 생성 건수 표시 |

### 2.4 경주 목록 필터·빈 상태

| 항목 | 상태 | 비고 |
|------|------|------|
| 경마장·날짜 필터 | 적용됨 | 지역(서울/제주/부산)·날짜 필터 있음 |
| 상태 필터 (예정/종료) | 적용됨 | 전체/예정/종료 선택, 클라이언트 필터 |
| 빈 상태 문구 | 적용됨 | 필터 적용 시 "선택한 조건에 맞는 경주가 없습니다", 미적용 시 "경주가 없습니다. 날짜를 선택하거나 KRA 동기화를 실행하세요." |

### 2.5 페이지별 보강 (우선 점검 권장)

| 페이지 | 부족할 수 있는 부분 |
|--------|----------------------|
| 경주 목록 | 날짜 기본값 KST 오늘 적용됨, 경마장·상태 필터, 빈 상태 |

| 예측 관리 | 목록 에러 재시도, 배치 ON/OFF·오늘 생성 건수·적중률 링크 적용됨 |
| 사용자 | 검색·에러 재시도 적용됨, 상세(구독·포인트·티켓 이력) 링크 |
| AI 설정 | 에러 시 다시 시도 적용됨 |

| 구독 관리·구독 플랜 | 에러 시 다시 시도 적용됨 |
| 통계·수익 대시보드 | 에러 시 다시 시도 적용됨 |
| 알림 관리·결제 내역 | 에러 시 다시 시도 적용됨 |
| AI 분석(analytics) | 에러 시 다시 시도 적용됨 |

## 3. 권장 진행 순서 (당장 손대기 좋은 것)

1. **WebApp:** 홈·경주 목록/상세 에러 시 "다시 시도" 적용됨. 빈 상태 문구 통일은 선택.  
2. **Admin:** 경주 목록·경기 결과 날짜 기본값 KST 오늘, 에러 시 다시 시도 적용됨.  
3. **공통:** 홈 prefetch 축소·매트릭스 중복 제거로 대용량 경고 완화 적용됨.
