## 1. WebApp (유저 사이트) 부족한 면

### 1.1 데이터·로딩·에러

| 항목 | 상태 | 비고 |
|------|------|------|
| 일부 목록/상세 에러 시 "다시 시도" 누락 | 적용됨 | 주요 페이지 DataFetchState·에러 시 다시 시도 적용 완료 |
| 빈 상태(Empty) 문구 통일 | 적용됨 | DataFetchState·emptyTitle "~가/이 없습니다" 패턴으로 페이지별 적용 |
| 대용량 페이지 데이터 경고 | 완화 적용 | 홈 prefetch limit 축소(results 60→30, races 50/150→30/40), 매트릭스 중복 prefetch 제거 |

### 1.2 UX·콘텐츠

| 항목 | 상태 | 비고 |
|------|------|------|
| 일일 로그인 보너스 | 적용됨 | 로그인 시 1일 1회 포인트 지급, 로그인 페이지에서 안내 메시지 |
| 연속 로그인 보상 (7일 연속 등) | 적용됨 | 7일 연속 시 RACE 티켓 1장, 프로필에 연속 일수 표시 |
| AI 신뢰도 표시 | 선택 | 예측별 confidence(low/medium/high), 모델 버전·방법론 노출 |
| 푸시 타이밍 개선 | 선택 | 첫 경주 30분 전 알림 등 |
| 이미지 최적화 | 선택 | 경주/말 사진 추가 시 next/image + alt (FEATURE_ROADMAP §5.4) |

### 1.3 모바일·접근성

| 항목 | 상태 | 비고 |
|------|------|------|
| Push Deep Link (모바일) | 서버 완료 | Expo deep link + WebView 연동 남음 |
| 터치 영역·버튼 크기 | 점검 필요 | 44px 최소 등 모바일 가이드 준수 여부 |
| 고대비/글자 크기 기본값 | 적용됨 | 설정에서 토글 가능 |

### 1.4 페이지별 보강 (우선 점검 권장)

| 페이지 | 부족할 수 있는 부분 |
|--------|----------------------|
| 홈 | 섹션별 에러 시 refetch, "오늘 경기 모두 종료" 외 문구 정리 |
| 경주 목록/상세 | 필터·URL 동기화, 결과 없음/예정 문구 |
| 결과 목록 | 날짜·경마장 필터, 빈 상태 |
| 종합 예측(매트릭스) | 날짜 변경 시 로딩·에러, 첫 방문 안내 |
| 마이페이지(구독·포인트·티켓) | 결제 실패/성공 안내, 만료 티켓 표시 |
| 프로필·설정 | 알림 설정 저장 시 "저장되었습니다" 인라인 피드백 적용됨, 회원탈퇴 플로우 |
| 말/기수/조교사 프로필 | DataFetchState 로딩·에러·빈 상태 적용됨 |

## 2. Admin (관리자) 부족한 면

### 2.1 공통

| 항목 | 상태 | 비고 |
|------|------|------|
| 로딩/에러/빈 상태 통일 | 적용됨 | 경주 목록·경기 결과 페이지 에러 시 메시지 + 다시 시도 버튼 추가 |
| 날짜 필터 KST | 적용됨 | 경주 목록·경기 결과·KRA 동기화 날짜 기본값 KST 오늘 적용 |
| 페이지별 권한/가드 | 점검 필요 | 관리자 로그인 필수 페이지 일괄 적용 (에러·재시도는 전 페이지 적용 완료) |

### 2.2 경주(KRA)·동기화

| 항목 | 상태 | 비고 |
|------|------|------|
| 경주 목록 날짜 기본값 | 적용됨 | KST 오늘 날짜를 기본값으로 설정 (admin 경주 관리) |
| 동기화 진행률·로그 | 적용됨 | SyncProgressBar, 토스트 메시지 |
| 결과 동기화 후 상태값 | 적용됨 | COMPLETED는 실제 결과/착순 반영 후만 (서버 수정 반영) |
| 배치 스케줄 요약 화면 | 적용됨 | Admin KRA 페이지에 배치 스케줄(예정/완료/실패) 테이블·집계 표시, 15초마다 갱신 |

### 2.3 사용자·구독·통계

| 항목 | 상태 | 비고 |
|------|------|------|
| 사용자 목록 검색·필터 | 적용됨 | 이메일·사용자명 검색 + 에러 시 다시 시도 |
| 구독/매출 대시보드 | 적용됨 | revenue, statistics 등 |
| 예측 정확도·티켓 사용량 | 예측 목록·티켓 사용 내역 에러 재시도 적용됨 | 경주별 적중률 링크는 선택 |

### 2.4 페이지별 보강 (우선 점검 권장)

| 페이지 | 부족할 수 있는 부분 |
|--------|----------------------|
| 경주 목록 | 날짜 기본값 KST 오늘 적용됨, 경마장·상태 필터, 빈 상태 |
| 경주 상세 | 에러 시 다시 시도 적용됨, 결과/출전표 요약·결과 수동 동기화 |
| KRA 동기화 | 일자 KST 오늘·동기화 로그 에러 재시도 적용됨, 이력 테이블 |
| 예측 관리 | 목록 에러 재시도 적용됨, 배치 on/off·오늘 생성 상태는 선택 |
| 사용자 | 검색·에러 재시도 적용됨, 상세(구독·포인트·티켓 이력) 링크 |
| AI 설정 | 에러 시 다시 시도 적용됨 |
| 설정(Config) | 시스템 설정·개별 구매 설정 에러 재시도 적용됨 |
| 구독 관리·구독 플랜 | 에러 시 다시 시도 적용됨 |
| 통계·수익 대시보드 | 에러 시 다시 시도 적용됨 |
| 알림 관리·결제 내역 | 에러 시 다시 시도 적용됨 |
| AI 분석(analytics) | 에러 시 다시 시도 적용됨 |
| 알림 | 발송 이력·실패 사유 |

## 3. 권장 진행 순서 (당장 손대기 좋은 것)

1. **WebApp:** 홈·경주 목록/상세 에러 시 "다시 시도" 적용됨. 빈 상태 문구 통일은 선택.  
2. **Admin:** 경주 목록·경기 결과 날짜 기본값 KST 오늘, 에러 시 다시 시도 적용됨.  
3. **공통:** 홈 prefetch 축소·매트릭스 중복 제거로 대용량 경고 완화 적용됨.
