# 용산종합지 스타일 승부예측 데이터 명세서

> `20260208VXO.pdf` (용산종합지)를 기준으로, **경주별 전문가 예상 비교 매트릭스**와 **전문가 코멘트**를 제공하는 UI 데이터 구조를 정의합니다.

---

## 1. 개요 (Overview)

용산종합지는 **표 + 전문가 예상**이 혼합된 형태로, **한눈에 여러 경주의 전문가 예측을 비교**할 수 있다는 장점이 있습니다.

- **참고**: 용산종합지 (20260208VXO.pdf), KRA API
- **대상**: 메인 종합 예상 화면, 전문가 코멘트 피드

---

## 2. 화면 구성 요소 (Component Structure)

### 2.1. 전체 경주 예상 매트릭스 (Prediction Matrix)

가로축은 전문가(AI 모델), 세로축은 경주(Race)로 구성된 대형 표.

| 경주 (Time) | 전문가 A | 전문가 B | 전문가 C | 전문가 D | AI 종합 (Consensus) |
|-------------|----------|----------|----------|----------|---------------------|
| 서울 1R (10:35) | 8 - 1 | 8 - 8 | 8 - 4 | 8 - 1 | **8** (축) |
| 서울 2R (11:00) | 7 - 1 | 6 - 9 | 11 - 3 | 9 - 1 | **1** |
| 서울 3R (12:05) | 4 - 4 | 4 - 6 | 4 - 11 | 4 - 4 | **4** |

**데이터 필드**

| 필드 | DTO/DB | 설명 |
|------|--------|------|
| raceId | race.id | 경주 식별자 |
| meet | meet, meetName | 경마장 (서울, 제주 등) |
| rcNo | rcNo | 경주 번호 (1R, 2R…) |
| stTime | stTime | 출발 시각 (10:35) |
| expertPredictions | { expertId: [1차, 2차] } | 전문가별 승부마 (복승 시 "8-1") |
| aiConsensus | 단일 마번 또는 "8-1" | AI 종합 추천 (축=1위) |

**JSON 예시**

```json
{
  "raceMatrix": [
    {
      "raceId": "1",
      "meet": "서울",
      "rcNo": "1",
      "stTime": "10:35",
      "predictions": {
        "expert_A": ["8", "1"],
        "expert_B": ["8", "8"],
        "expert_C": ["8", "4"],
        "ai_consensus": "8"
      },
      "consensusLabel": "축"
    }
  ]
}
```

**UI 컨셉**

- 엑셀 스타일 그리드 또는 가로 스크롤 카드
- 핀치 줌(Pinch Zoom) 지원 권장
- 마번 셀에 게이트 색상 적용 (`RACE_DETAIL_UI_SPEC.md` 참조)

---

### 2.2. 오늘의 승부처 & 추천마 (Hot Picks & Commentary)

전문가들의 한 줄 평과 추천 마번을 카드/말풍선 형태로 표시.

| 전문가 (Source) | 대상 경주 | 추천 마번 | 분석 코멘트 |
|----------------|-----------|-----------|-------------|
| 차민수의 히든카드 | 서울 1R | **8 (콜미스카이)** | "조교 강도 대폭 올려 승부 의지 강력함." |
| 박건우 (신마뉴스) | 서울 2R | **6 (비트코드)** | "막판 근성 보일 수 있는 선추입형. 전력 상승세 뚜렷." |
| 강병준 (정보력 1위) | 서울 10R | **9 (매직포션)** | "부담중량 적절하고 최근 걸음이 가벼움. 강력 추천." |

**데이터 필드**

| 필드 | 설명 |
|------|------|
| expertId / expertName | 전문가 식별자/이름 |
| raceId, meet, rcNo | 대상 경주 |
| hrNo, hrName | 추천 마번, 마명 |
| comment | 분석 코멘트 텍스트 |
| keywords | 강조 키워드 (조교 강도, 선추입형, 부중 적절 등) |

**UI/UX 포인트**

- 키워드 강조: 볼드 또는 `#태그`로 핵심 표현 시각화
- 추천 마번: 게이트 색상 배지로 표시
- 피드 형태: 인스타그램 스타일 스크롤

---

### 2.3. 적중 내역 (Hit Record)

지난주/어제 적중 이력을 상단에 노출하여 신뢰도 강화.

| 필드 | 설명 |
|------|------|
| hitDate | 적중일 (YYYY-MM-DD) |
| description | "퍼펙트 적중! 1월 17일 토요일 전경주 적중" |
| details | "서울-11, 제주-6" 등 상세 내역 |

**UI 표현**

- 상단 롤링 배너(Ticker) 또는 공지 배너

## 3. 탭 구조 (Layout)

| 탭 | 내용 | 대상 사용자 |
|----|------|-------------|

## 4. UI/UX 디자인 가이드 (Design Implementation)

### 4.1. 색상 시스템 (신문지 감성)

| 용도 | 색상 | hex |
|------|------|-----|
| Primary (강조, 적중, 우승마) | 적색 | #D93025 |
| Secondary (복승, 2위) | 청색 | #1A73E8 |
| Background (종이 질감) | 연회색 | #F8F9FA |
| Text | 진한 회색 | #202124 |

> 기존 테마(`globals.css`)와 충돌하지 않도록 CSS 변수로 오버라이드 가능하게 구성 권장.

### 4.2. 인터랙션 (Interaction)

| 동작 | 반응 |
|------|------|
| 마번 클릭 | 해당 말 상세 정보 모달 (이전 성적, 기수 등) |
| 전문가 필터 | "차민수", "강병준" 등 특정 전문가만 표시 |
| 경주 클릭 | 해당 경주 상세 페이지로 이동 |

## 5. API/DTO 요구 사항

### 5.1. 예상 매트릭스 API

```
GET /api/predictions/matrix?date=2026-02-08&meet=서울
```

**응답 구조**

- `raceMatrix`: 경주별 전문가 예상 배열
- 전문가 수, 경주 수는 가변

### 5.2. 전문가 코멘트 API

```
GET /api/predictions/commentary?date=2026-02-08&limit=20&offset=0
```

**응답 구조**

- `comments`: 전문가별 코멘트 배열 (전문가명, 경주, 마번, 마명, 코멘트, 키워드)

### 5.3. 적중 내역 API

```
GET /api/predictions/hit-record?limit=5
```

## 7. 비즈니스: 종합 예상표 + 예측권 (검토 중)

종합 예상표는 **별도 예측권/종합예상권** 판매를 고려할 수 있음.

| 옵션 | 설명 |
|------|------|
| A. 기존 예측권 재사용 | "오늘 종합 예상" = 예측권 1장 사용 (일 할당) |
| B. 별도 상품 | "종합 예상권" — 일/주 단위 구매 상품 |
| C. 구독 전용 | PREMIUM 구독자만 종합 예상표 무제한 |

**현재 구현**: 로그인 필요. 예측권 게이트는 추후 적용 예정.
