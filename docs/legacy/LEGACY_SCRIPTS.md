# 📜 스크립트 사용 가이드

## 🚀 과거 데이터 초기화

OddsCast AI 모델 학습을 위한 과거 경마 데이터를 수집합니다.

### 사용법

#### 1. 데이터 커버리지 확인

```bash
npm run init:data:check
```

현재 DB에 저장된 데이터의 통계를 확인합니다.

#### 2. 최근 30일 데이터 수집 (기본)

```bash
npm run init:data
# 또는
npm run init:data:30days
```

#### 3. 특정 연도 전체 수집

```bash
npm run init:data:year  # 2024년 전체
# 또는 다른 연도
npm run init:data -- --year=2023
```

#### 4. 특정 기간 수집

```bash
npm run init:data -- --start=20240101 --end=20241231
```

#### 5. 최근 N일 수집

```bash
npm run init:data -- --days=90  # 최근 90일
npm run init:data -- --days=365  # 최근 1년
```

### 📊 예상 데이터량

| 기간 | 예상 경주 수 | 예상 소요 시간 |
| ---- | ------------ | -------------- |
| 30일 | ~1,500 경주  | ~30분          |
| 90일 | ~4,500 경주  | ~1.5시간       |
| 1년  | ~18,000 경주 | ~6시간         |
| 3년  | ~54,000 경주 | ~18시간        |

### ⚙️ 기술 세부사항

#### 수집 데이터

1. **경주 결과** (API4_3)
   - 경주 기본 정보
   - 순위 및 기록
   - 마필/기수/조교사 정보

2. **경주 계획** (API72_2)
   - 예정된 경주 정보
   - 경주 조건 및 상금

3. **출전표** (API26_2)
   - 상세 출전 정보O
   - 마필 상태 데이터

4. **확정 배당율** (API160)
   - 배당률 정보
   - 승식별 데이터

#### API 호출 제한

- **TPS**: 초당 10건
- **일일 한도**: 10,000건
- **딜레이**: 요청 간 1초 대기

### 🔧 문제 해결

#### "Table doesn't exist" 에러

```bash
# DB 초기화
npm run db:init
```

#### "API rate limit" 에러

- 딜레이를 늘리거나 배치 크기를 줄입니다
- 스크립트를 여러 번 나눠서 실행합니다

#### 중복 데이터

- 자동으로 중복 체크하여 업데이트합니다
- 기존 데이터는 유지됩니다

### 💡 권장 사용 시나리오

#### 초기 설정 (처음 1회)

```bash
# 1. DB 확인
npm run init:data:check

# 2. 최근 1년 데이터 수집
npm run init:data -- --year=2024

# 3. 커버리지 재확인
npm run init:data:check
```

#### 일상적 업데이트

```bash
# 매주 월요일 최근 7일 데이터 업데이트
npm run init:data -- --days=7
```

#### AI 학습용 대량 수집

```bash
# 3년치 데이터 (AI 모델 학습용)
npm run init:data -- --start=20220101 --end=20241231
```

### ⚠️ 주의사항

1. **서버 실행 필요**: MySQL이 실행 중이어야 합니다
2. **네트워크**: KRA API 접근 가능해야 합니다
3. **시간 소요**: 대량 데이터는 시간이 오래 걸립니다
4. **중단**: Ctrl+C로 중단 가능 (이미 수집된 데이터는 유지됨)
